Preset Ecosystem DNA

Not all TSConfigs are created equal. MyTSFy tailors the compiler options based on where your code will run. This radar chart visualizes the "profile" of each preset, showing the trade-offs between browser compatibility, server capabilities, and abstraction levels.

Vite Preset

Frontend Focus

Optimized for bundlers. High browser compatibility focus, delegates module resolution to tools like Vite/Rollup.

Node Preset

Backend Focus

Modern backend (Node 18+). Uses NodeNext for native ESM/CJS support. Strict server-side typing.

Next.js Preset

Hybrid Framework

Framework specific. Preserves JSX, uses hybrid resolution, and prioritizes incremental build speeds.

Under the Hood

The MyTSFy CLI makes opinionated choices to ensure your project uses modern standards. Here we analyze the output configurations to show exactly what's changing between presets.

Target JS Freshness

Comparing the ECMAScript target version. Higher means more modern code output.

Insight: Node uses ES2022 (very modern), Vite uses ES2020 (broad modern support), while Next.js defaults to ES5 (maximum compatibility, handled by SWC later).

Module Resolution Strategy

The most confusing part of TS. How does MyTSFy handle imports for each stack?

Insight: bundler is the new standard for web apps, removing the need for extension hacks. NodeNext is critical for modern backend libraries.
